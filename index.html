GIS GE example clients

Following links are to the different example clients which can be used for verifying GIS GE basic functionality. Purpose of these clients is to be educational and give an idea how GIS GE can be used for creating 3D scenes.Requirement for succesfully running these examples requires that GIS GE server has been configured as described in the Installation guide and in the Unit testing plan. Used test asset in the examples is elevation data from the square area which one side 6km long. Web client's scenemanager in all examples work so that whole test asset area (bounding box) is divided to 5x5 grid, and new grid block is dynamically loaded regarding camera position in the terrain. First block to be loaded is lower left corner of the terrain layer.

Camera control in the example clients
Camera can be moved with keyboard by using arrow -keys or "A,S,D,W" -keys. Camera elevation can be increased by pressing space -key, and descreased with "C" -key. Oriantation can be changed by pressing and keeping mouse left button pressed and moving mouse.

Example clients:

  • XML3D object example client
    • Example client creates queries which response is 3D object definition of the queried terrain area. Client places received XML3D definition directly to the DOM tree, which leads rendering terrain object to the display.
    • When server is properly configured, 3D building models can be placed to the terrain with selecting them in Select objects -section.
    • Test asset PostGis data contains different level of details information. This can be tested bu using Set Level Of Details list, LOD values between 4-10 are used.

  • Asset instance example client
    • This example demonstrates GIS GE capability to create and provide 3D terrain object files, which can be referenced directly from the DOM instead of adding whole object definition to the DOM tree. Benefit of this approach is cleaner DOM treee even with complicated scenes.

  • Example client for DEM Geotiff usage as for source for 3D terrain
    • Here is demonstration of using grascale DEM model as for source to 3D terrain creation. Data response is in octet-stream format, which contains elevation points for the queried area. This data is used as a source for creatin 3D terrain.

  • Octet-stream example client
    • Example client queries elevation data which response format is octet-stream. This data is used as a source for creatin 3D terrain.
